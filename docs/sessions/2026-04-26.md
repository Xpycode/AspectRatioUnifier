## Progress

### Wave 7 — TargetSizeStrategy picker shipped

Branch: `feature/wave-7-target-strategy` (started this session, **not yet committed**).

- New enum `TargetSizeStrategy { min, median, max }` in `Services/RatioTargetResolver.swift` with `.label` and `.help` per case
- `RatioTargetResolver.resolve(bucket:sources:strategy:)` learned all three:
  - **Median** = `bucket.medianSize` (current behavior, preserved)
  - **Min** = inscribed-rectangle math: for each included image, largest target-ratio rect that fits without upscale; take min across all. Math: `max(target.w/source.w, target.h/source.h)` is the scale factor; inscribed-rect height = `min(source.h, source.w/T)` then width derived to preserve T.
  - **Max** = same math, max instead of min
- `AppState.targetSize` now dispatches on `targetSizeStrategy`. Sources = all included images (post-exclusion), not just the picked bucket — Min/Max are global properties of the export set, only Median is bucket-local. Asymmetry is intentional.
- `HistogramView.readout` got a 3-segment Picker (`.segmented` + `.controlSize(.small)`) under "Target committed", with per-segment `.help()` tooltips
- Default = **Min** (decision 2026-04-25: photographer-cleaning-archives mindset)

### Crop overlay bug fix (CropPreviewBadge)

Found while testing Wave 7 with a 1.43:1 target. Dashed crop rectangles were drawn against the full badge frame, but the image inside renders with `.aspectRatio(.fit)` into a smaller letterboxed sub-region — so the overlay bled past the visible image edges, especially when source aspect (e.g. 2:3 portrait) diverged from container aspect.

Fix: new `fitRect(sourceAspect:in:)` mirrors SwiftUI's `.fit` math, then `cropOverlay` places the dashed rectangle inside the image's actual rendered bounds. ~15 LOC in `Views/CropPreviewBadge.swift`. Latent bug for a while; only obvious with extreme aspect mismatches.

### Grid sort feature shipped

User requested sort-by-resolution after experiencing the Min "hunt for outliers" workflow. Constraint: can't find the binding image by sorting on the wrong axis (Height sort doesn't surface portrait-binding outliers).

- New file `Models/GridSort.swift`: `GridSortDimension { ratio, width, height, megapixels }` enum + `GridSort { dimension, reversed }` struct (Equatable, Codable). `apply(to:)` does the sort with stable id-string tiebreak.
- `ImageItem` gained `.aspectRatio` and `.megapixels` computed properties (in `AppState.swift` next to `originalSize`)
- `AppState.gridSort` stored property with UserDefaults `didSet` persistence — same pattern as `ExportSettings.userProfiles`
- `ImageGridView.visibleImages` now calls `appState.gridSort.apply(to:)` instead of inline ratio sort (6 lines → 1 line)
- New `gridSortMenu` computed view in `ContentView`, placed in `.primaryAction` toolbar group before the help/add/trash buttons. `Menu` containing a `Picker` (auto-checkmark on selection, canonical Finder/Lightroom pattern) + Divider + Toggle for "Reverse Order"
- Default sort: aspect ratio ascending (preserves prior behavior — no first-launch surprise)
- **Persistence policy**: sort persists across launches (muscle memory), `targetSizeStrategy` does NOT (opinionated default of Min should be rediscovered per launch)

### SF Symbol bug — `arrow.down.arrow.up` doesn't exist

Tried to flip the sort-menu icon between `arrow.up.arrow.down` (normal) and `arrow.down.arrow.up` (reversed). The latter isn't a real SF Symbol — SwiftUI silently rendered an empty image, which collapsed the Menu's hit region, making the entire sort button disappear.

Fix: hold the icon constant. Reversed state is communicated via the open menu's Toggle row + tooltip text update. Lesson saved for the future: verify symbol names against the SF Symbols app before conditional swaps.

## Decisions to record

- 2026-04-26: Target-size strategies — three options (Min / Median / Max), default Min. Min/Max are global over the included image set; Median is bucket-local.
- 2026-04-26: Min/Max use inscribed-rectangle math (largest target-ratio rect that fits inside each source, take min/max of resulting heights with width derived from ratio).
- 2026-04-26: Grid sort — four dimensions (Aspect Ratio, Width, Height, Megapixels) + a single reversed toggle. Persisted to UserDefaults. Default Aspect Ratio ascending.
- 2026-04-26: Picker-in-Menu pattern adopted for sort UI (canonical macOS, used by Finder/Lightroom). `.menuIndicator(.hidden)` + `.menuStyle(.borderlessButton)` to match adjacent toolbar buttons visually.
- 2026-04-26: CropPreviewBadge must compute its overlay against the `.fit`-rendered image bounds, not the full badge frame. Latent bug, only obvious at extreme aspect mismatches.
